# FitLog 종목 기록 방식(type) 설계 — 시간 종목·유산소 종목 지원

날짜: 2026-07-17
상태: 승인됨

## 배경 / 목적

현재 모든 종목은 무게(kg)·횟수·RPE 세 칸으로만 기록할 수 있다. 사용자는 다음을 원한다:

1. 등 부위의 "매달리기"는 kg·횟수 없이 **초(시간)만** 기록.
2. **유산소** 부위를 신설하고 **런닝 / 천국의 계단 / 사이클** 3종목을 등록.
   유산소 종목은 **km · 시간(분:초) · 칼로리** 세 항목을 기록.

## 설계 결정 요약

- 종목마다 **기록 방식(type)** 속성을 부여: `'weight'`(근력, 기본) | `'time'`(시간) | `'cardio'`(유산소).
- 이름 기반 하드코딩(예: 이름이 "매달리기"면 시간 취급)은 이름 변경 시 깨지므로 채택하지 않음.
- 유산소 시간 입력은 **분:초 두 칸** (사용자 선택).
- 종목 추가/수정 모달에서 기록 방식을 **직접 선택** 가능 (사용자 선택).

## 데이터 구조

### 종목 DB (`fitlog_exerciseDB`)

```
exercise: { id, name, type }
```

- `type`: `'weight' | 'time' | 'cardio'`. **없으면 `'weight'`로 간주** → 기존 데이터 완전 호환.

### 세션 (`fitlog_sessions`)

세션 내 종목 항목에 type 스냅샷을 저장한다:

```
exercise: { name, type, sets }
```

- 종목을 세션에 추가하는 시점에 종목 DB에서 type을 조회해 저장한다.
  종목 DB에서 종목이 삭제/변경돼도 과거 기록은 기록 당시 방식으로 표시된다.
- 기존 세션 데이터에는 type이 없음 → `'weight'`로 간주.

### 세트 형태 (type별)

| type | 세트 객체 | 표시 예 |
|---|---|---|
| weight | `{ weight, reps, rpe }` (현행 유지) | `1세트 60kg × 8회 RPE 8` |
| time | `{ durationSec }` | `1세트 45초` |
| cardio | `{ km, durationSec, kcal }` (각 항목 null 허용) | `1세트 5.2km · 30:30 · 320kcal` |

- cardio 표시 시 null인 항목은 생략한다.
- durationSec은 초 단위 정수로 저장(입력은 분:초 두 칸, 저장 시 `분*60+초`).

## 자동 반영 (마이그레이션)

앱 초기화 시(`seedIfEmpty` 이후) 1회 실행되는 마이그레이션:

1. `fitlog_exerciseDB`에 이름이 **유산소**인 부위가 없으면 생성하고
   **런닝 / 천국의 계단 / 사이클** 3종목을 `type: 'cardio'`로 등록.
   유산소 부위가 이미 있으면 그 안에 없는 종목만 추가.
2. 종목 DB 전체에서 이름이 **매달리기**인 종목의 type을 `'time'`으로 설정.
3. 변경이 있을 때만 저장. 멱등(여러 번 실행돼도 결과 동일).
4. `js/data.js`의 PRESET에도 유산소 부위를 추가(신규 사용자 시드용).
   PRESET 구조는 type을 표현할 수 있게 확장하되 기존 부위는 weight 기본값 유지.

기존에 kg×횟수로 기록된 매달리기 세트는 **수정하지 않는다**. 세트 표시는
세트 객체가 가진 필드 기준으로 포맷하므로 예전 기록은 예전 형식 그대로 보인다.

## 기록 화면 (운동 탭, `js/workout.js`)

### 세트 입력 칸 (추가 행·수정 행 공통)

- **weight**: 현행 유지 (kg / 횟수 / RPE / 버튼).
- **time**: `초` 한 칸 + 버튼.
- **cardio**: `km` / `분` / `초` / `kcal` 네 칸 + 버튼.

### 검증 규칙

- time: 초는 1 이상의 정수.
- cardio: **최소 한 항목 입력** 필요. 입력된 항목만 검증:
  - km: 0보다 큰 숫자(소수 허용)
  - 분: 0 이상의 정수, 초: 0~59 정수 (분·초 중 하나라도 입력되면 시간으로 인정)
  - kcal: 1 이상의 정수
  - 빈 항목은 null로 저장.

### 지난번(prev) 표시

- weight: 현행 유지(최고 무게 세트).
- time: 지난번 세션의 최장 시간 — `지난번: 60초`.
- cardio: 지난번 세션의 마지막 세트를 그대로 — `지난번: 5km · 30:00 · 300kcal`.

### 프리필

현행 로직 유지(마지막 세트 → 없으면 지난번 기록), type별 필드만 다르게 채운다.

## 종목 추가/수정 화면 (`js/exercises.js`)

- 종목 추가·이름 수정 모달에 **기록 방식 선택**(근력 / 시간 / 유산소) 추가. 기본값 근력.
- 부위 추가/수정 모달은 변경 없음.
- 기록 방식을 바꿔도 과거 세션 기록은 스냅샷 type으로 표시되므로 영향 없음.

## 통계 탭 (`js/stats.js`)

- **1RM 추이**: 종목 목록에 세션 종목의 type 스냅샷이 weight(또는 type 없음)인
  종목만 포함 — 종목 DB 조회 없이 세션 데이터만으로 동작. 1RM 계산 시
  weight 필드가 없는 세트는 건너뛴다.
- **운동 볼륨**: 근력 세트만 집계. (time/cardio 세트는 weight가 없어 현행 계산식에서도
  0으로 더해지지만, 명시적으로 weight 필드 존재 여부로 걸러 의도를 드러낸다.)
- 유산소 전용 통계(주간 거리 등)는 이번 범위 제외.

## 범위 밖 (하지 않음)

- 유산소 전용 통계/차트.
- 기존 매달리기 기록(kg×횟수)의 데이터 변환.
- localStorage 키 추가(4개 유지), export/import 스키마 버전 변경
  (새 필드는 기존 import 검증을 그대로 통과).

## 검증 방법

- `node --check js/*.js` 문법 확인.
- 브라우저 수동 확인:
  1. 기존 데이터가 있는 상태로 열어 유산소 부위·3종목 자동 생성, 매달리기 시간 타입 전환 확인.
  2. 매달리기 세트: 초만 입력·표시·수정·삭제.
  3. 런닝 세트: km/분/초/kcal 입력(일부만 입력 포함)·표시·수정·삭제.
  4. 근력 종목 기록이 현행과 동일하게 동작.
  5. 통계 탭: 1RM 목록에 유산소/시간 종목 미노출, 볼륨 정상.
  6. 루틴에 유산소 종목 포함 → 기록 탭 적용 시 cardio 입력 칸 표시.
  7. export → import 왕복 후 데이터 동일.
