# FitLog 설계 문서

날짜: 2026-07-04
원본 요구사항: `헬스기록앱_PRD.md`

## 1. 개요

웹 브라우저에서 서버/로그인 없이 동작하는 개인용 헬스 트래커.
Vanilla JavaScript + HTML + CSS, `localStorage` 저장, 외부 의존성 없음.
모바일 우선 반응형, 다크모드 기본 스타일. 로컬 파일(`file://`)로 열어도 동작해야 한다.

## 2. 파일 구조

```
헬스앱/
├── index.html          # 4개 탭 화면의 마크업 골격
├── css/style.css       # 다크모드 기본, 모바일 우선 반응형
└── js/
    ├── data.js         # 기본 부위/종목 프리셋 데이터 상수
    ├── storage.js      # localStorage 읽기/쓰기, 프리셋 시딩, export/import
    ├── ui.js           # 탭 전환, 토스트, 모달 등 공용 UI 유틸
    ├── exercises.js    # 부위/종목 커스텀 관리 (추가/수정/삭제)
    ├── workout.js      # [기록] 탭: 월 달력, 세션, 계층형 종목 선택, 세트 입력
    ├── body.js         # [체중] 탭: 입력, 리스트, SVG 그래프
    ├── stats.js        # [통계] 탭: 1RM 추이, 볼륨 추이 (SVG)
    ├── routines.js     # [루틴] 탭: 템플릿 CRUD, 적용
    └── app.js          # 초기화, 각 모듈 연결
```

- 일반 `<script>` 태그로 순서대로 로드 (ES 모듈 미사용 — `file://` 호환성 우선).
- 렌더링: 각 탭마다 render 함수를 두고, 데이터 변경 시 해당 탭만 다시 그린다.
  전역 상태는 각 모듈의 지역 변수 + localStorage로 관리.

## 3. 데이터 모델

localStorage 키 4개. 모든 쓰기는 입력 즉시 반영(자동 저장).

### `fitlog_sessions` — 운동 세션 배열
```js
{
  id: "uuid",
  date: "2026-07-03",        // YYYY-MM-DD, 세션은 날짜당 1개
  routineName: "가슴/삼두",   // 선택
  exercises: [
    {
      name: "벤치프레스",
      sets: [ { weight: 60, reps: 10, rpe: 7 } ]  // rpe는 선택
    }
  ]
}
```

### `fitlog_bodyMetrics` — 체중 기록 배열
```js
{ date: "2026-07-03", weight: 72.5, bodyFat: 18.2 }  // bodyFat 선택
```

### `fitlog_routines` — 루틴 템플릿 배열
```js
{ id: "uuid", name: "가슴/삼두 day", exerciseNames: ["벤치프레스", ...] }
```

### `fitlog_exerciseDB` — 부위/종목 목록
```js
{
  bodyParts: [
    { id: "uuid", name: "가슴",
      exercises: [ { id: "uuid", name: "벤치프레스" } ] }
  ]
}
```

- 최초 실행 시(`fitlog_exerciseDB` 부재 시) 기본 프리셋 자동 시딩:
  가슴/등/어깨/팔/하체/복근 6개 부위, 부위별 대표 종목 4~6개.
- 1RM 추정: Epley 공식 `weight × (1 + reps / 30)`.

## 4. 탭별 기능

### 4.1 [기록] 탭

**월 달력 (탭 최상단)**
- `◀ 2026년 7월 ▶` 헤더로 이전/다음 달 이동.
- 운동 세션이 있는 날은 칸을 악센트 색으로 표시, 오늘은 테두리로 구분.
- 날짜 클릭 → 아래 세션 영역이 그날 기록으로 전환. 과거/미래 상관없이
  오늘과 동일하게 세트 추가/수정/삭제 가능(선택 날짜 기준으로 세션 열림).
- 기록 없는 날짜 클릭 시 빈 상태에서 바로 기록 추가 가능.

**세션 영역**
- 기본 선택 날짜는 오늘. 선택 날짜에 세션이 없으면 첫 종목 추가 시 생성.
- 종목 추가: 계층형 선택 UI — "부위 선택" 화면 → 부위 탭 → "종목 선택" 화면
  → 종목 탭 → 세트 입력. 전 과정 타이핑 없이 탭만으로 가능.
- 부위/종목 편집: 두 선택 화면 각각에 "+ 추가", "수정", "삭제" 제공.
  편집 결과는 즉시 선택 UI와 `fitlog_exerciseDB`에 반영.
  부위 삭제 시 확인 후 하위 종목도 함께 삭제(기존 세션 기록은 유지).
- 세트 기록: 무게(kg)/횟수/RPE(선택) 입력. 세트 추가 시 직전 세트 값 자동 채움.
- 이전 기록 비교: 같은 종목이 있는 가장 최근 과거 세션에서
  "지난번: 60kg × 10" 형태로 표시(최고 무게 세트 기준).
- 세트/종목 삭제 및 수정 가능.

### 4.2 [체중] 탭
- 날짜별 체중(+선택적 체지방률) 입력. 같은 날짜 재입력 시 덮어쓰기.
- 최근 기록 리스트(최신순), 항목 삭제 가능.
- 기간별(7일/30일/90일/전체) 꺾은선 그래프 — SVG 직접 구현.

### 4.3 [통계] 탭
- 종목 선택 드롭다운 → 해당 종목의 최고 무게(1RM 추정) 추이 꺾은선 그래프.
- 주간/월간 총 볼륨(무게×횟수 합계) 추이 그래프.
- 빈도 캘린더는 [기록] 탭 월 달력으로 통합되어 통계 탭에는 두지 않는다.

### 4.4 [루틴] 탭
- 루틴 템플릿 생성/수정/삭제. 종목 구성은 계층형 선택 UI 재사용.
- [기록] 탭에서 "루틴 불러오기" → 템플릿 선택 시 해당 종목 목록이
  현재 세션에 자동 추가(세트는 비어 있는 상태).

### 4.5 데이터 관리
- 전체 데이터 JSON export(파일 다운로드) / import(파일 선택).
- import 시 구조 검증 후 전체 교체. 실패 시 기존 데이터 유지 + 오류 토스트.
- export→import 왕복 시 데이터 100% 복원.

## 5. 공용 UI / 비기능

- SVG 꺾은선 그래프 공용 함수 1개로 체중/1RM/볼륨 그래프 재사용.
- 토스트: 저장/삭제/오류 피드백. 모달: 이름 입력, 삭제 확인.
- 입력 유효성: 음수/빈 값/숫자 아님 차단. RPE는 1~10 범위.
- 빈 상태 UI: 기록/체중/통계/루틴 각 탭에 안내 문구.
- 터치 영역 최소 44px, 모바일 하단 탭 네비게이션.
- localStorage 파싱 실패 시 해당 키만 기본값으로 복구(다른 데이터 보존).

## 6. 구현 단계 (PRD 5단계 플랜, 단계별 사용자 확인)

1. **레이아웃 스캐폴딩** — 4탭 네비게이션, 빈 화면, 다크모드 반응형 골격.
2. **운동 기록 핵심** — 프리셋 시딩, 계층형 선택, 부위/종목 편집, 세트 기록,
   월 달력(날짜 선택/색칠/과거 수정).
3. **체중/통계** — 체중 입력·그래프, 1RM·볼륨 그래프. 더미 데이터 주입
   스크립트(콘솔 실행용)로 검증.
4. **루틴/데이터 관리** — 템플릿 CRUD·적용, JSON export/import.
5. **QA/폴리싱** — 빈 상태, 유효성, 토스트, 디자인 통일, 터치 영역 점검.

각 단계 완료 시 브라우저에서 직접 확인 후 다음 단계 진행.
