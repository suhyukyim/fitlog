# 헬스 기록앱 (FitLog) PRD

## 1. 개요

**앱 이름(가칭):** FitLog
**목적:** 웹 브라우저에서 별도 서버/로그인 없이 운동 기록, 체중, 신체 변화를 기록하고 추이를 시각화하는 개인용 헬스 트래커
**기술 스택:** Vanilla JavaScript + HTML + CSS, `localStorage` 기반 데이터 저장 (외부 의존성 없음)
**타겟 사용자:** 헬스장에서 웨이트 트레이닝을 하며 세트/무게/횟수를 기록하고 싶은 개인 사용자 (본인)

---

## 2. 핵심 기능

### 2.1 운동 기록 (Workout Log)
- 날짜별 운동 세션 생성
- 종목 추가 시 타이핑 없이 **계층형 선택**으로 종목 지정
  - 1단계: 부위 선택 (가슴 / 등 / 어깨 / 팔 / 하체 / 복근 등)
  - 2단계: 해당 부위에 속한 종목 선택 (예: 가슴 → 벤치프레스, 인클라인 덤벨프레스...)
  - 예: `운동 종류 → 가슴 → 벤치프레스` 순서로 탭하여 선택
- 종목별로 세트 단위 기록: 무게(kg) / 횟수(reps) / RPE(선택)
- 이전 세션의 같은 종목 기록을 자동으로 불러와 비교 표시 ("지난번: 60kg x 10")
- 세트/종목 삭제 및 수정

### 2.1.1 부위/종목 커스텀 관리 (Body Part & Exercise Manager)
- 기본 제공되는 부위/종목 목록(프리셋)은 앱 최초 실행 시 자동으로 세팅됨
- 사용자가 자유롭게:
  - 부위 추가/이름 수정/삭제
  - 특정 부위 안에 종목 추가/이름 수정/삭제
- 이 목록은 [기록] 탭의 계층형 선택 UI와 연동되어, 여기서 편집한 내용이 즉시 선택지에 반영됨
- 별도 관리 화면(예: 설정 내 "운동 종목 관리") 또는 계층 선택 UI 안에 "+ 새 종목 추가" 옵션으로 구현

### 2.2 체중 및 신체 기록 (Body Metrics)
- 날짜별 체중 입력
- (선택) 체지방률, 골격근량 등 추가 지표
- 기간별 체중 변화 그래프

### 2.3 통계 및 시각화 (Progress Dashboard)
- 종목별 최고 무게(1RM 추정치) 추이 그래프
- 주간/월간 총 운동 볼륨(무게 x 횟수 합계) 추이
- 최근 7일/30일 운동 빈도 캘린더 뷰

### 2.4 루틴 템플릿 (Routine Templates)
- 자주 하는 루틴(예: "가슴/삼두 day")을 템플릿으로 저장
- 템플릿 선택 시 종목 목록 자동 세팅, 세트 기록만 하면 되도록

### 2.5 데이터 관리
- localStorage 기반 저장 (기기 종속)
- JSON 형식으로 내보내기(export) / 가져오기(import) — 백업 및 기기 이전용

## 3. 데이터 모델 (예시)

```js
// 운동 세션
{
  id: "uuid",
  date: "2026-07-03",
  routineName: "가슴/삼두",
  exercises: [
    {
      name: "벤치프레스",
      sets: [
        { weight: 60, reps: 10, rpe: 7 },
        { weight: 65, reps: 8, rpe: 8 }
      ]
    }
  ]
}

// 체중 기록
{
  date: "2026-07-03",
  weight: 72.5,
  bodyFat: 18.2 // optional
}

// 루틴 템플릿
{
  id: "uuid",
  name: "가슴/삼두 day",
  exerciseNames: ["벤치프레스", "인클라인 덤벨프레스", "케이블 크로스오버", "트라이셉스 익스텐션"]
}

// 부위/종목 목록 (커스텀 관리 대상)
{
  bodyParts: [
    {
      id: "uuid",
      name: "가슴",
      exercises: [
        { id: "uuid", name: "벤치프레스" },
        { id: "uuid", name: "인클라인 덤벨프레스" }
      ]
    },
    {
      id: "uuid",
      name: "등",
      exercises: [
        { id: "uuid", name: "데드리프트" },
        { id: "uuid", name: "랫풀다운" }
      ]
    }
  ]
}
```

## 4. 비기능 요구사항
- 모바일 브라우저에서도 사용 가능한 반응형 레이아웃 (헬스장에서 폰으로 기록)
- 오프라인 완전 동작 (외부 API/CDN 의존 최소화)
- 빠른 입력 UX: 세트 추가 시 이전 세트 값 자동 채움
- 데이터 손실 방지를 위한 자동 저장(입력 즉시 localStorage 반영)

---

## 5. Claude Code 5단계 프롬프트 플랜

### 1단계 — 레이아웃 스캐폴딩
```
Vanilla JS/HTML/CSS로 "FitLog" 헬스 기록앱의 기본 레이아웃을 만들어줘.
- 하단 탭 네비게이션: [기록] [체중] [통계] [루틴]
- 각 탭에 대응하는 빈 화면(placeholder) 구성
- 모바일 우선 반응형 레이아웃, 깔끔한 다크모드 기본 스타일
- localStorage 연동은 아직 하지 말고 UI 뼈대만 완성
완료 기준: 4개 탭 전환이 정상 동작하고, 각 화면의 레이아웃이 모바일/데스크톱에서 깨지지 않아야 함.
```

### 2단계 — 운동 기록 핵심 기능 (계층형 종목 선택 포함)
```
1단계에서 만든 [기록] 탭에 아래 기능을 구현해줘.

[부위/종목 기본 데이터]
- 앱 최초 실행 시 기본 부위(가슴/등/어깨/팔/하체/복근 등)와 부위별 기본 종목을 localStorage에 자동 세팅

[종목 추가 - 계층형 선택 UI]
- 종목 추가 버튼 클릭 시 "부위 선택" 화면 → 부위 탭하면 "해당 부위 종목 선택" 화면으로 이동
- 종목을 탭하면 선택 완료, 세트 입력 화면으로 전환
- 타이핑 없이 전부 탭으로 선택 가능해야 함

[부위/종목 자유 편집]
- 부위 선택 화면과 종목 선택 화면 각각에 "+ 추가", "수정", "삭제" 기능 제공
- 부위 추가/이름수정/삭제, 부위 내 종목 추가/이름수정/삭제 가능
- 편집한 내용은 즉시 선택 UI에 반영되고 localStorage에 저장

[세트 기록]
- 종목별 세트 추가: 무게/횟수/RPE 입력
- 같은 종목의 직전 기록을 자동으로 불러와 "지난번: OOkg x OO회" 형태로 표시
- 모든 데이터는 localStorage에 즉시 저장

완료 기준:
- 운동 종류 → 부위 → 종목 순서로 탭만으로 종목을 선택할 수 있어야 함
- 부위/종목을 추가·수정·삭제하면 선택 UI와 데이터에 즉시 반영되어야 함
- 세션-부위-종목-세트 계층 구조가 정확히 저장/조회되며, 새로고침 후에도 데이터가 유지되어야 함
```

### 3단계 — 체중 기록 및 통계
```
[체중] 탭과 [통계] 탭을 구현해줘.
- 체중 탭: 날짜별 체중(및 선택적으로 체지방률) 입력, 최근 기록 리스트, 기간별 꺾은선 그래프
- 통계 탭: 종목 선택 시 해당 종목의 최고 무게 추이 그래프, 주간/월간 총 볼륨 추이, 최근 30일 운동 빈도 캘린더
- 그래프는 외부 라이브러리 없이 Canvas 또는 SVG로 직접 구현
완료 기준: 최소 2주 이상의 더미 데이터로 그래프가 정확한 추이를 보여줘야 함.
```

### 4단계 — 루틴 템플릿 및 데이터 관리
```
[루틴] 탭과 데이터 백업 기능을 구현해줘.
- 루틴 템플릿 생성/수정/삭제 (자주 쓰는 종목 세트를 이름으로 저장)
- [기록] 탭에서 템플릿 선택 시 종목 목록 자동 세팅
- 설정 화면(또는 별도 버튼)에서 전체 데이터 JSON export/import 기능
완료 기준: 템플릿 적용 시 종목이 정확히 채워지고, export한 JSON을 import했을 때 데이터가 100% 복원되어야 함.
```

### 5단계 — 최종 QA 및 폴리싱
```
전체 앱을 점검하고 다듬어줘.
- 빈 상태(empty state) UI 처리 (기록 없을 때 안내 문구)
- 입력 값 유효성 검사 (음수/빈 값 방지)
- 로딩/저장 시 시각적 피드백(토스트 등)
- 전체적인 디자인 통일성 점검 (색상, 여백, 폰트 크기)
- 모바일 터치 영역 크기 점검
완료 기준: 실제 사용 시나리오(운동 세션 생성→기록→통계 확인→루틴 적용) 전체가 오류 없이 매끄럽게 동작해야 함.
```
